# Milestone 8 — Workbook revamp: gentler floor + reward-as-check

Built on top of M7 (caregiver TOTP MFA). The frozen pre-flight contract
is `docs/specs/M8.md`; this file is the implementation plan and records
implementation-time decisions that deviate from or extend that contract.
Part B is backed by **ADR 0004** (workbook reward delivered as a check),
which reverses the CLAUDE.md data-model note + rule 5.

M8 has two parts:

- **Part A — lower the skill floor.** Add kindergarten + grade 1 to the
  workbook content system. Additive, no ADR.
- **Part B — reward becomes a deposited check.** The finished-work
  reward is a `kind: "check"` deposit code the patient deposits through
  the existing "Deposit a Check" flow; the typed-reward-code
  `/submit-work` path is removed. ADR 0004.

## Gate

**Coding does not start until M7 is closed** (its Step 8 production smoke
+ green-check). The spec is frozen; this plan can be written and reviewed
in parallel, but the first code edit waits on M7 close.

## Resolved (from the spec, locked at freeze 2026-06-05)

Full text in `docs/specs/M8.md` § Resolved:

1. **Grade encoding — numeric `0 | 1 | 2 | 3`, K = 0.** Smallint
   unchanged; numeric comparisons keep working; "Kindergarten" / "Grade
   1" shown at the caregiver UI edge only.
2. **Reward friction — consolidate as-is.** Reward = `kind: "check"`
   code, deposited via the existing `DepositWizard` (photo → amount →
   8-char code), **unchanged**. No wizard changes.
3. **Back-compat — migrate.** One-time migration converts unused
   `kind = "workbook"` codes to `kind = "check"`; `"workbook"` enum value
   retained for historical (used) rows; no new `workbook` codes minted.
4. **One milestone.** Parts A and B ship together as M8.

## Implementation-time decisions recorded here

### One deposit_codes row per workbook, `kind = "check"` (2026-06-05)

A workbook today is a single `deposit_codes` row carrying both the
reward (`amountCents`) and the printed content
(`workbookKind` / `workbookGrade` / `contentSeed`). M8 keeps **one row**
and only flips its `kind` from `"workbook"` to `"check"`. The row *is*
the reward check and still carries the workbook content for the PDF +
answer-key views. Rejected the two-row split (one content record + one
check) — it would orphan the existing codeId-keyed PDF/answers lookups
for no benefit.

### Surface separation moves from `kind` to workbook-ness (2026-06-05)

Because workbook rewards become `kind = "check"`, `kind` alone no longer
separates the caregiver's **Checks** surface from its **Workbooks**
surface. The discriminator becomes the presence of workbook content:

- **Workbooks** list / PDF / answers / auto-title count: filter
  `isNotNull(depositCodes.workbookKind)` (was `eq(kind, "workbook")`).
- **Checks** list / PDF: filter `eq(kind, "check") AND
  isNull(workbookKind)` (was `eq(kind, "check")`) so plain checks and
  workbook rewards don't bleed into each other.

This is the load-bearing change in Part B; every `kind = "workbook"`
read in the tree must be audited. Known call sites:
`workbooks/page.tsx:100`, `workbooks/[codeId]/pdf/route.ts:51`,
`workbooks/actions.ts` (auto-title count ~line 104), the answers route,
and `checks/page.tsx` + `checks/[codeId]/pdf/route.ts:43`.

### Reward check renders as the final page of the workbook PDF (2026-06-05)

Spec Open Question 2b (deferred to this plan): the reward check is the
**last page of the existing workbook PDF**, not a separate printable.
One artifact, one "Open workbook" action; the caregiver prints once and
the patient deposits the last page. `renderWorkbookPdfStream`
(`lib/workbook-pdf`) gains a final check page, reusing the layout from
`renderCheckPdfStream` (`lib/check-pdf`) — extract the shared check
drawing into something both can call rather than duplicating it.

### Data-only migration `0005` (2026-06-05)

The back-compat migration changes **data, not schema** (the `"workbook"`
enum value is retained), so `drizzle-kit generate` produces no diff.
Add `drizzle/0005_*.sql` as a hand-authored data migration:

```sql
update public.deposit_codes
  set kind = 'check'
  where kind = 'workbook' and status = 'unused';
```

Mirror the `0004` journal/snapshot pattern: a data-only migration's
snapshot is identical to `0004`'s. Confirm the project's apply mechanism
(`pnpm db:push`) walks the file before committing the mechanics. This is
the one place M8 hand-authors SQL — justified because there is no schema
diff for drizzle-kit to generate.

## Scope and decisions

### Step 1 — widen `WorkbookGrade`

`lib/workbook-content/types.ts`: `WorkbookGrade = 0 | 1 | 2 | 3`. Add a
caregiver-facing label helper (e.g. `gradeLabel(g)` →
`"Kindergarten" | "Grade 1" | "Grade 2" | "Grade 3"`) co-located in the
content lib and re-exported from `index.ts`. The `workbookGrade`
smallint already holds 0–3; no schema change. `isWorkbookSample`'s
`typeof v.grade === "number"` check stays valid. No pause — Step 2
depends on it.

### Step 2 — author K + grade-1 problem banks

New JSON under `lib/workbook-content/grade-0/` (K) and
`lib/workbook-content/grade-1/`, one file per category
(`math-facts.json`, `word-problems.json`, `reading-passages.json`,
`sequencing.json`, `simple-logic.json`), matching the grade-2/3 shapes
exactly. Wire both into `BANKS` in `index.ts`.

**Minimum bank sizes** (so the seeded sampler never starves — each bank
must hold ≥ the cumulative single-workbook draw for its category across
all `PAGE_SHAPES`): math-facts **≥ 22**, word-problems **≥ 6**,
reading-passages **≥ 3**, sequencing **≥ 2**, simple-logic **≥ 4**.
Author comfortably above these.

**Content guidance:** K = single-digit recognition / counting / +1 / very
short concrete sentences; grade 1 = sums/differences within 20, two-step
sequences, 2–3 sentence passages. Gentle, concrete, dignity-preserving
(the audience is an adult with dementia, not a child — phrasing must
never feel like children's homework; keep it neutral and respectful).

`bank-sanity.test.ts`: extend to assert the new grades meet the minimums
for every category. `sampler.test.ts`: add a grade-0 and grade-1 sample
case (deterministic, no duplicates within a workbook).

**Pause** — the user reviews the K/1 content before we wire it into the
UI. Content quality is the entire point of Part A; a bad floor is worse
than no floor.

### Step 3 — grade picker UI + validation

- `workbooks/page.tsx` grade `<select>`: add `<option value="0">
  Kindergarten (gentlest)</option>` and `<option value="1">Grade 1
  </option>` above the existing Grade 2/3; keep `defaultValue` as the
  gentlest sensible default (consider defaulting to `"1"` or `"0"` given
  the feedback — caregiver can raise it).
- `workbooks/actions.ts` `createWorkbookSchema.grade`:
  `z.enum(["0", "1", "2", "3"]).transform(v => Number(v) as 0|1|2|3)`.
- The "Previously generated" table renders `w.workbookGrade ?? "—"` raw
  — swap to `gradeLabel(w.workbookGrade)` so 0 shows "Kindergarten", not
  "0".

No pause — Part A is now complete and shippable on its own.

### Step 4 — data migration `0005`

Add `drizzle/0005_*.sql` per the decision above (unused
`workbook` → `check`). Verify against a local DB that used workbook rows
are untouched and unused ones flip. No pause.

### Step 5 — generation mints `check` + surface filter switch

- `workbooks/actions.ts` `createWorkbookAction`: insert with
  `kind: "check"` (was `"workbook"`); keep `workbookKind` /
  `workbookGrade` / `contentSeed` / `label` / `targetAccountId`. Switch
  the auto-title count query from `eq(kind, "workbook")` to
  `isNotNull(workbookKind)`. Audit-log `actionKind` stays
  `workbook_code_generated`.
- **Workbooks** surfaces — `workbooks/page.tsx`,
  `workbooks/[codeId]/pdf/route.ts`, the answers route: replace
  `eq(depositCodes.kind, "workbook")` with
  `isNotNull(depositCodes.workbookKind)`.
- **Checks** surfaces — `checks/page.tsx`,
  `checks/[codeId]/pdf/route.ts`: tighten `eq(kind, "check")` to also
  require `isNull(workbookKind)` so workbook rewards don't appear in the
  plain-checks list/PDF.
- Update the workbooks page copy: it currently says the patient "enters
  the code on their 'Submit Completed Work' screen" — change to describe
  depositing the check via "Deposit a Check." The reward still appears as
  the printed check on the last page.

No pause — Step 6 finishes the artifact.

### Step 6 — reward check on the workbook PDF's final page

`lib/workbook-pdf` `renderWorkbookPdfStream`: append a final page that
renders the reward as a real check (payee = patient display name, amount
= `amountCents`, the 8-char `code`, memo/label), reusing the check
drawing from `lib/check-pdf`. Factor the shared check layout into a
component both PDFs import rather than duplicating. The patient tears off
the last page and deposits it. No pause.

### Step 7 — remove the typed reward-code path

Delete `app/(patient)/patient/[id]/submit-work/**` entirely
(`page.tsx`, `actions.ts` with `lookupWorkbookCodeAction` /
`redeemWorkbookCodeAction`, `CodeEntryForm.tsx`, `confirm/`, `done/`).
Audit for inbound links — `patient/[id]/page.tsx` references submit-work
(grep `submit-work` across the patient group) — remove or repoint them.
Confirm the patient app still falls back **calmly** (home screen, no
404) for a stale `/submit-work` URL, per the patient-UX rule. Remove any
now-dead tests referencing the removed actions.

**Pause** — the user confirms the patient flow end-to-end on a workbook:
generate → print → deposit the reward check via "Deposit a Check" → the
old `/submit-work` URL falls back calmly.

### Step 8 — CLAUDE.md edit + close-out

- **CLAUDE.md** (per ADR 0004): update the data-model note
  (`deposit_codes ... kind (check|workbook)`) and architecture rule 5
  ("single-use codes for printed checks **and** workbooks") to reflect
  that workbook rewards are now delivered *as* checks. Keep the
  historical `"workbook"` enum mention if useful, but the live rule is
  checks-only. This edit lands here, with the code, so the rules file
  never contradicts the tree.
- **Green-check:** `pnpm typecheck && pnpm lint && pnpm test --run`
  green; `pnpm build` clean; forbidden-patient-vocab grep over the
  patient route group clean (no "grade"/"kindergarten"/"workbook"
  leaking into patient UI).

**Pause** — M8 close-out review.
